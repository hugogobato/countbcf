Adjust how to deal with number of forests coz while for multibart.cpp we are dealing with two forests (namely mu and tau forest), for countbart.cpp, we already have three different forests for zero inflated models, namely f, f0, and f1 with f0 being used to estimate the likelihood of an observation to be in the zero inflated part of the DGP, f1 being used to estimate the likelihood of an observation to not be in the zero inflated part of the DGP, and f being used to estimate the count value of the observation given it is not part of the zero inflated part. For countbcf.cpp, we would keep these functions f, f0 and f1, but now instead of one BART model for each, we would have two BART models equal to the mu and tau forest of the BCF model. And they should be updated the same way as for the BCF model, namely, we run the MCMC algorithm for the mu forest (using the observations y) and once we are done we run it for the tau forest (using the residual y-mu as if it was the outcome variable and only using the treated observations given the BCF formula is Y = mu(X)+tau(X)*Z). It is worth mentioning that the MCMC part to update kappa, Z_i, xi, and phi continues the same since once again we are not changing the underlying structure and still having f, f0 and f1.
